密钥与地址仅写入当前浏览器本地存储；公用电脑请勿填写。密钥随请求发往您填写的代理；代理需在服务端开启 TRUST_CLIENT_KEYS=1 才会使用（公开展示的共享代理请勿开启）。

DeepSeek（与官方 OpenAI 兼容格式，见 DeepSeek API 文档）：

  1. 不想部署 Worker：选择「直连 DeepSeek」，填写并保存 API 密钥即可（请求发往 api.deepseek.com）。若浏览器拦截跨域，再改回「自建代理」。
  2. 自建代理：选择「自建代理」，在「POST 地址」填 Cloudflare Worker 的 HTTPS（如 https://xxx.workers.dev），不要填 api.deepseek.com。
  3. （仅自建代理）Worker 环境变量（控制台或 wrangler）： UPSTREAM_CHAT_URL 填官方示例之一： https://api.deepseek.com/chat/completions 或 https://api.deepseek.com/v1/chat/completions； OPENAI_MODEL 填 deepseek-chat（或文档中的 deepseek-reasoner 等）。
  4. （仅自建代理）密钥二选一：在 Worker 用 wrangler secret put OPENAI_API_KEY 存入 DeepSeek 的 Key；或在页面「API 密钥」填写并保存，且 Worker 设置 TRUST_CLIENT_KEYS=1（仅自用，公开展示勿开）。
